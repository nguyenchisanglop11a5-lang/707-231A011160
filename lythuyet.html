1. Sự khác biệt giữa position: absolute và position: relative

position: relative: phần tử vẫn chiếm không gian trong flow bình thường của trang, nhưng có thể dịch chuyển so với vị trí gốc bằng top/right/bottom/left. Dùng để điều chỉnh layout mà không phá vỡ cấu trúc chung, thích hợp với responsive design. 

.container { position: relative; }
.child { position: relative; top: 10px; left: 5px; }

position: absolute: phần tử được đặt chính xác so với phần tử cha gần nhất có position khác static. Nó không chiếm không gian trong flow bình thường. Dùng cho tooltip, popup, menu overlay. 

.container { position: relative; }
.tooltip { position: absolute; top: 0; left: 0; }

Ảnh hưởng khi deploy: Absolute cần phần tử cha position tương ứng để responsive; Relative dễ thích ứng với các thiết bị khác nhau.

2. Quy trình đăng ký domain & cấu hình hosting

Quy trình:

  1. Đăng ký domain: chọn nhà cung cấp domain (GoDaddy, Namecheap...)
  2. Chọn hosting: GitHub Pages, Netlify, Vercel (không cần server riêng)
  3. Cấu hình DNS: trỏ tên miền tới IP của hosting (A record, CNAME)
  4. Upload code website (HTML/CSS/JS) lên hosting
  5. Kiểm tra website trên domain

Vai trò của DNS: Chuyển tên miền thành IP của server hosting, giúp trình duyệt truy cập website thông qua domain.

Công cụ hỗ trợ: GitHub Pages, Netlify. Chúng cho phép deploy nhanh mà không cần server riêng.

Rủi ro bảo mật: Nếu cấu hình sai DNS, website không truy cập được hoặc dễ bị tấn công; cần HTTPS, hạn chế lưu thông tin nhạy cảm trực tiếp trên JS.

3. So sánh let, const và var trong JavaScript

  • var: function-scope, hoisting lên đầu function, dễ gây lỗi khi truy cập trước khi khai báo.
  • let: block-scope, không hoisting, dùng khi giá trị thay đổi.
  • const: block-scope, không thể gán lại, dùng cho hằng số hoặc object/array không thay đổi tham chiếu.

Ví dụ lỗi thường gặp:

// var: hoisting
console.log(x); // undefined
var x = 5;

// let: truy cập trước khai báo
console.log(y); // ReferenceError
let y = 10;

// const: gán lại
const z = 3;
z = 4; // TypeError

Debug: Sử dụng Chrome DevTools → Sources → Breakpoints → Console để theo dõi biến và lỗi hoisting.

4. Event Delegation trong JavaScript

Event Delegation là kỹ thuật gán 1 event listener lên phần tử cha để xử lý các phần tử con, đặc biệt hữu ích cho danh sách động (dynamic list). Giúp giảm bộ nhớ, tăng hiệu suất và dễ quản lý sự kiện. 

document.getElementById('list').addEventListener('click', e => {
  if(e.target.tagName === 'LI'){
    alert('Clicked: ' + e.target.textContent);
  }
});

Lợi ích khi kết hợp HTML/CSS: - Không cần thêm listener cho từng item → giảm DOM overhead. - Tối ưu CSS transitions/animations cho item động. - Giữ layout responsive, dễ bảo trì.
